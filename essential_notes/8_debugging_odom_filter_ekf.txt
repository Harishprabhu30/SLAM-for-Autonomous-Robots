Debugging Steps:

This behavior tells us that your EKF node (ekf_filter_node) has started but isn’t actually publishing /odometry/filtered.

-- Step 1: 

✅ /odom → publishing correctly (confirmed)
❌ /odometry/filtered → not being published yet
⚠️ That means EKF node is running but not producing output, usually due to one of the following:

| Possible Issue                      | Explanation                                                                     |
| ----------------------------------- | ------------------------------------------------------------------------------- |
| Topic mismatch                      | The input topic name in `ekf.yaml` doesn’t match the actual `/odom` topic name  |
| TF frame mismatch                   | The frame IDs (`odom`, `chassis`) in your config don’t match your Isaac TF tree |
| No valid input messages received    | EKF node doesn’t publish until it gets a valid first input                      |
| Parameter path or YAML format issue | Node loaded but parameters not properly applied                                 |

---------------------------------
-- Step 2:

Let’s Check if the EKF Node Is Alive

Run this:

ros2 node list


✅ You should see something like:

/ekf_filter_node


If you don’t, your launch file may not have loaded it correctly.

Then check if it’s subscribing:

ros2 node info /ekf_filter_node


You should see something like:

Subscribers:
  /odom
  /imu (optional)
Publishers:
  /odometry/filtered
  /tf
  /diagnostics


If you don’t see /odom under Subscribers, that means your EKF config isn’t connected correctly to your odometry topic.
